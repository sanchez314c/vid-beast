## Contributing to Documentation

### Improving Documentation
- **Fix typos and errors**: Submit a pull request
- **Add missing information**: Create an issue or PR
- **Suggest improvements**: Open an issue with suggestions
- **Translate**: Help translate documentation to other languages

### Documentation Style
- Use **clear, concise language**
- Include **code examples** for technical content
- Add **cross-references** to related documents
- Follow **consistent formatting** and structure

### Review Process
1. **Create** a documentation branch
2. **Make** your changes
3. **Test** links and examples
4. **Submit** a pull request
5. **Review** and incorporate feedback
